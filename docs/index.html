// architecture

No build step. No framework. Just files.

A Python build script parses your markdown notes into a JSON index. A static HTML page searches it with Fuse.js. That's it.

system architecture
01

Markdown Parser

Python build.py recursively scans /notes/ subdirectories, extracts procedures from ## headers, steps from ### headers, code blocks, tags, and variables into structured JSON.

02

Static Frontend

Single index.html with vanilla JavaScript. No framework, no bundler, no transpilation. Fuse.js for search, Highlight.js for syntax coloring.

03

Variable Engine

Detects <variable> patterns in code blocks. Workspace system saves variable sets in localStorage. Values substitute on copy -- never modifies the source.

04

Offline First

Service worker caches all assets on first visit. Works fully offline. Zero CDN dependencies. Everything ships in the site/ folder.

// quickstart

Running in 30 seconds

Clone, build, serve. No Docker, no npm, no configuration files.

bash
# Clone the repository
$ git clone https://github.com/Real-Fruit-Snacks/Surge.git
$ cd surge
# Build the search index
$ python3 build.py
# Start local server
$ python3 -m http.server 8080 --directory site
# Or use any static server:
$ npx http-server site -p 8080     # Node.js
$ php -S localhost:8080 -t site   # PHP

Open http://localhost:8080 in your browser. Your markdown notes from /notes/ are now searchable. Press v to open the variable panel, define your targets, and start copying commands with values pre-filled.

Add your own notes to /notes/<category>/ using standard markdown with ## headers for procedures and ### headers for steps. Run build.py again to rebuild the index.
